(main.dart) 
. contains firebase starting logics 

(app.dart) 

. here check connectivity status , Internet connectivity status
. it contains native theme features , lib/Theme/nativeTheme.dart
. if not than go to (InternetLostPage) , lib/widgets/NetworkLostWidget.dart 
. if connected go to (SplashScreen) , lib/appScreen/splashScreen.dart

(SplashScreen) lib\appScreen\splashScreen.dart

. here we check location permission using checkLocationPremission function 
    - if isLocationServiceEnabled is true than run checkGPS function 
      where we check location permission having enum values [denied, deniedForever, whileInUse, always, unableToDetermine]

    - if isLocationServiceEnabled is false than run showLocationAlert function 
      where it shows a dialog which want us to aalow location permission having accept and dany button

. than we run _getTokenFromFirebaseMessaging here we get token from firebase

. than after we run _init function 
    - it defines a duration of 1 second+, until this it shows a logo image After the specified duration, it executes a function using a Timer
      It checks connectivity using the br.checkConnectivity() if the device is connected to the internet:
      It retrieves the device token using Firebase Cloud Messaging (FirebaseMessaging.instance.getToken()).

    - It retrieves data and base URL from shared preferences and If user data is available in shared preferences:
      if data != null than it checks It checks the user type (mCurrrentUserModel.type)

      If the user type is "ASP" than it navigates to the technician dashboard screen using Navigator.pushReplacement.
      else  it navigates to the dealer main dashboard screen.

      if data == null it navigates to the login screen. 

      If the device is not connected to the internet, it displays a network error snack bar using showNetworkErrorSnackBar function.

(LoginWithUsernameScreen) lib\appScreen\technician\Login\LoginWithUsernameScreen.dart

. _getTokenFromFirebaseMessaging() it Fetch Firebase messaging token initially 
. initPlatformState it collect the information of our android / IOS mobile device and store into deviceData data
. Now Build function runs it have an userId , password having controllerPhone and controllerPassword
. forgot password button which navigate to (ForgotPasswordScreen) lib\appScreen\dealer\registration\ForgotPassword\ForgotPasswordScreen.dart
. Create account button which navigate to (SignupScreen) lib\appScreen\dealer\registration\SignupScreen.dart

. On Submit button 
    - TechnicianappBaseUrl() API calls lib\models\businessLayer\queries.dart
    - FLow after User clicks on the submit button.
        onTap is triggered : 
            It checks if isUserID and isPassword are both true.
            If valid, calls _loginValidateUpdate.

        In _loginValidateUpdate :   
            if controllerPhone.text is not empty.  calls runMutation with the phone number as an argument.

        GraphQL Mutation Execution:
            The MutationWrapper's Mutation widget executes the mutation.
            Mutation widget's onCompleted callback is set to _onCompletedUpdate.

        Handling Mutation Result:
            _onCompletedUpdate is called with the mutation result.
            Checks the status in the result.
            If status is 200, 
                updates global URLs, 
                saves to shared preferences, 
                and calls the appropriate login function (DealerAppLoginwithBaseUrl or TechnicianappLoginwithBaseUrl).
                
            If status is not 200, clears shared preferences and shows an error message.

(ForgotPasswordScreen) lib\appScreen\dealer\registration\ForgotPassword\ForgotPasswordScreen.dart

. dealerForgotPassword() API calls lib\models\businessLayer\queries.dart
. Data Flow
    User enters User ID: The user enters their ID in the TextField controlled by controllerPhone.
    Submit button clicked: When the submit button is clicked, _loginValidateUpdate is called.
    Run Mutation: This function calls the runMutation provided by the MutationWrapper.
    GraphQL Mutation: The mutation dealerForgotPassword is executed on the server.
    Handle Response: The response is handled in the onCompleted callback. If successful, the OTP input field is displayed and a timer is started.
    
    when yourOTPvisible = true  
        A Card with input OTP field and a varify button shown 
        and after clicking on varify button calls ChangePasswordScreen() lib\appScreen\dealer\registration\ForgotPassword\ChangePasswordScreen.dart

    enableResend = true 
        after finishing timer of 60 sec it restart the same process.
